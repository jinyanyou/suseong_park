========================================
카카오맵 타일 렌더링 버그 정리
========================================
날짜: 2026-06-26

----------------------------------------
1. 문제
----------------------------------------
- 페이지 첫 진입 시 카카오맵 타일(지도 이미지)이 표시되지 않고 흰 화면만 나옴
- 카카오 로고와 축척 바는 정상 표시 (SDK 로드, 맵 객체 생성은 성공)
- 목록에서 공원을 클릭하면 그때서야 지도가 정상 표시됨
- 동 필터 탭 중 일부(전체, 고산1동, 고산2동, 고산3동, 두산동, 만촌1동)를
  클릭하면 지도가 다시 흰 화면으로 변함
- 나머지 동 필터는 정상 작동

----------------------------------------
2. 원인
----------------------------------------
map.setBounds() 메서드의 연쇄 호출이 타일 로딩을 방해함.

[상세 원인]
(1) render()가 초기 로드 시 여러 번 연쇄 호출됨
    - xlsx 파일 로드 완료 후 render()
    - custom-parks.json 로드 완료 후 render()
    - coord-fixes.json 로드 완료 후 render()
    - park-overrides.json 로드 완료 후 render()
    - kakao.maps.load() 콜백 내 render()

(2) 각 render() 호출이 renderMap() -> map.setBounds()를 실행
    - setBounds()는 내부적으로 center와 level을 동시에 재계산
    - 기존 타일 요청을 전부 취소하고 새 타일을 요청

(3) 타일 로딩 중에 setBounds()가 다시 호출되면
    - 이전 타일 요청이 취소됨
    - 새 타일 요청이 시작되지만 또 다시 setBounds()가 호출됨
    - 타일이 한 번도 완전히 로드되지 못하고 흰 화면으로 남음

[특정 동에서만 실패한 이유]
- 동 필터 클릭 시 타이밍상 비동기 render()가 겹쳐서
  setBounds()가 중복 실행되는 경우가 해당 동에서 발생

[공원 클릭 시 정상 작동한 이유]
- focusPark() 함수는 setBounds()를 사용하지 않고
  map.setCenter() + map.setLevel()을 사용
- 이 방식은 단순히 뷰포트를 이동/줌하는 것이라 타일 로딩을 방해하지 않음

----------------------------------------
3. 해결
----------------------------------------
map.setBounds()를 완전히 제거하고
map.setCenter() + map.setLevel()로 대체함.

- 표시된 공원들의 좌표 평균값으로 center 계산
- 좌표 분산(spread)에 따라 적절한 level 자동 결정
  - spread > 0.08 -> level 7 (구 전체)
  - spread > 0.04 -> level 6
  - spread > 0.02 -> level 5
  - spread > 0.005 -> level 4
  - 그 외 -> level 3 (동 단위)

이 방식은 focusPark()에서 공원 클릭 시 사용하던 것과
동일한 패턴이므로 안정적으로 작동함.
